## 🧐 Conceptual Overview
EvoScientist is a self-evolving multi-agent framework that moves beyond static research. It uses a "Human-on-the-loop" paradigm where AI agents continuously improve their research strategies through **Persistent Memory**.

## 🚀 Unique "Evolutionary" Features

### 1. Persistent Research Memory
*   **Ideation Memory:** Stores historical "research directions" (what hypotheses worked/failed).
*   **Experimentation Memory:** Stores code-level successes (which processing functions actually improved accuracy).
*   **Result:** The agent never repeats the same scientific mistake twice.

### 2. The Evolution Manager Agent (EMA)
*   **The Logic:** A dedicated high-level agent that audits the " interaction history" of the Coder and Researcher.
*   **The Result:** It distills "Learned Wisdom" into strategies, increasing the success rate of subsequent experiments.

### 3. Specialization of Roles
*   Splits the workflow into **Planner, Researcher, Coder, Debugger, Analyst, and Writer**.
*   This mimics a real-world research organization, leading to higher precision and "Scientific Taste."

## 🤝 Potential "Evolutionary" Applications for LiveMirror

### 1. Persistent Wisdom Layer
*   Implement `EVOLUTIONARY_MEMORY.json` to store which modality weighting (Sentiment/Intent/Audio) is most predictive during specific market regimes (e.g., High Volatility vs. Crab Market).

### 2. Multi-Agent Research Board
*   Transition **SelfMirror** from a single agent to a specialized team:
    - **Researcher:** Proposes weight adjustments.
    - **Coder:** Implements in the Sandbox.
    - **Analyst:** Compares backtest results.
    - **EMA:** Saves the strategy to Persistent Memory.
